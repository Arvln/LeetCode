# 1658.Minimum-Operations-to-Reduce-X-to-Zero

從數組兩側最少挑幾個元素可達到目標和，稍微模擬一下，概念很熟悉，回顧 #918 求環形數組最大和也是類似套路，表面上問數組兩側，題目做熟掌握正難則反的思維，思路會往數組中間的子數組和上靠。只要是子數組和，幾乎都可用前綴和解，這邊一樣把數組擴大成原本兩倍並算出前綴和，這題只要純粹紀錄前綴和索引，不用維護滑窗最值，用一個哈希表來記錄就行。特別注意更新答案的條件，要將當前元素索引限制在大於等於原數組長度n、前一個元素索引小於等於n，如此一來才是數組兩側元素構成的子數組，最基本的子數組長度小於等於n也要帶上。子串問題常用技巧還有滑動窗口，將題目轉成求和為總和減去目標和的子數組最長為多少，就是滑窗常用應用。
